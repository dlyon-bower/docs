**Shipped 2026-04-30 · Permissions**

Some workspaces are highly collaborative. Others — especially solo researchers
or labs handling sensitive IP — want everything private by default, and to
explicitly choose what to share.

We now support both.

## Private Mode

Workspace owners can switch a workspace into **Private Mode** from
**Settings → Workspace → Privacy**. With Private Mode on:

- New artifacts (notes, attachments, collections, projects) start **visible
  only to you**.
- The "everyone in the workspace" default is replaced with explicit shares —
  you decide who sees each artifact.
- The workspace member list still exists; what changes is the *default*
  visibility of new content.

Existing artifacts keep their current visibility — Private Mode only affects
what gets created from this point on. You can change any individual artifact's
visibility at any time using the share dialog.

## Workspace owner controls

Workspaces now have a dedicated **Owner** role with admin-only access to:

- Privacy mode toggle
- Workspace billing
- Member management (invite / remove / change role)
- Workspace instructions for Bird (the AI-readable workspace memory)

Other roles (**Manager**, **Editor**, **Viewer**) keep their existing
capabilities.

## Why this matters

Until now, Bower's defaults assumed a shared lab. That's right for most teams,
but it left solo researchers and IP-sensitive labs needing to manually lock
down every new note. Private Mode flips the default; explicit shares replace
implicit access. It's the same model Notion ships for sensitive workspaces.

## Try it

- **Existing workspace:** Settings → Workspace → Privacy → toggle Private Mode.
- **New workspace:** the privacy mode picker is now part of workspace creation.
